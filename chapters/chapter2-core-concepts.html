üîß Chapter 2: Core Concepts

Understanding the building blocks of ECharts visualizations

Anatomy of an ECharts Option

Every ECharts visualization is defined by an option object. Understanding this structure is crucial for creating effective charts. Let's break down each component:

Interactive Option Structure

üéØ Interactive Option Builder

Experiment with the ECharts option structure in real-time:

1. Title Component

The title component defines the main heading and subtitle of your chart:

title: {
  text: 'Sales Overview',           // Main title
  subtext: 'Q1 2024',              // Subtitle
  left: 'center',                  // Position: 'left', 'center', 'right'
  top: 'top',                      // Position: 'top', 'middle', 'bottom'
  textStyle: {                     // Title styling
    fontSize: 18,
    fontWeight: 'bold',
    color: '#333'
  }
}

2. Tooltip Component

Tooltips provide interactive information when users hover over chart elements:

tooltip: {
  trigger: 'axis',                 // 'item', 'axis', 'none'
  axisPointer: {                   // Axis indicator
    type: 'shadow'                 // 'line', 'shadow', 'cross'
  },
  formatter: function(params) {    // Custom tooltip content
    return params[0].name + ': ' + params[0].value;
  }
}

3. Legend Component

Legends help users identify different data series:

legend: {
  data: ['Sales', 'Profit', 'Cost'],  // Series names
  orient: 'horizontal',               // 'horizontal', 'vertical'
  left: 'center',                     // Position
  top: 'bottom'                       // Position
}

4. Axis Components

Axes define the coordinate system for your data:

Axis Types Comparison

// Category Axis (X-axis for bar charts)
xAxis: {
  type: 'category',                // 'category', 'value', 'time', 'log'
  data: ['Jan', 'Feb', 'Mar'],     // Category labels
  name: 'Month',                   // Axis name
  nameLocation: 'middle',          // Name position
  nameGap: 30                      // Distance from axis
}

// Value Axis (Y-axis for bar charts)
yAxis: {
  type: 'value',                   // Numeric values
  name: 'Amount ($)',              // Axis name
  min: 0,                          // Minimum value
  max: 1000,                       // Maximum value
  interval: 200                    // Tick interval
}

5. Series Component

The series component contains your actual data and defines the chart type:

Series Types Overview

series: [{
  name: 'Sales',                   // Series name
  type: 'bar',                     // Chart type
  data: [120, 200, 150, 80],      // Data values
  itemStyle: {                     // Item styling
    color: '#4a90e2'
  },
  emphasis: {                      // Hover effects
    itemStyle: {
      color: '#357abd'
    }
  }
}]

6. Data Formats

ECharts supports multiple data formats. Here are the most common:

Data Format Examples

// Simple array
data: [120, 200, 150, 80]

// Array of objects
data: [
  { value: 120, name: 'Jan' },
  { value: 200, name: 'Feb' },
  { value: 150, name: 'Mar' }
]

// 2D array for scatter plots
data: [[120, 200], [180, 300], [250, 400]]

// Object with multiple series
data: {
  '2023': [120, 200, 150],
  '2024': [180, 250, 200]
}

7. Chart Container & Initialization

Proper initialization is crucial for ECharts to work correctly:

// HTML container with fixed dimensions
<div id="main" style="width:600px;height:400px;"></div>

// JavaScript initialization
var chart = echarts.init(document.getElementById('main'));

// Apply options
chart.setOption(option);

// Handle window resize
window.addEventListener('resize', function() {
  chart.resize();
});

üö® Critical Requirements

  • Container must exist: The DOM element must be present before calling echarts.init()
  • Fixed dimensions: Container must have explicit width and height
  • Single initialization: Call init() only once per container
  • Resize handling: Always handle window resize for responsive charts
